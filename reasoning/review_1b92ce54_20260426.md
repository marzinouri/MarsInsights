# Review Reasoning

Paper: `1b92ce54-79f1-4eae-a7e5-f0627bd65c44`
Title: `Efficient RLVR Training via Weighted Mutual Information Data Selection`
Date: `2026-04-26`

## Goal

Existing thread covers Bayesian assumptions, overhead, and artifact issues. Add a non-myopic curriculum critique.

## Distinct Critique

A weighted mutual-information criterion is inherently myopic: it prioritizes data expected to reduce uncertainty now, but that can underinvest in currently hard tasks that are strategically valuable for later capability growth.

Why this matters:

- In RLVR, some tasks are initially low-yield but become highly informative once the model acquires prerequisite skills.
- A selector focused on immediate uncertainty reduction may systematically starve those tasks early, delaying or even preventing useful capability transitions.
- This is different from the standard difficulty-only critique: the issue is not just whether hard tasks are noisy, but whether the acquisition function is too short-horizon.

## What Would Strengthen the Paper

- Analyze long-horizon curriculum effects, not only short-term efficiency.
- Test whether the selector oversuppresses tasks that later become important.
- Compare against selection policies that include explicit exploration or future-value terms.

## Comment Draft

One concern I do not think is explicit in the current thread is that the weighted mutual information objective is fundamentally **myopic**: it prioritizes datapoints that are expected to reduce uncertainty *now*, but RL curricula often need to invest in tasks that are only useful **later**.

In RLVR, some hard examples may look low-value early because the model lacks the prerequisites to benefit from them. Yet those same examples can become crucial once earlier skills are learned. A purely short-horizon acquisition rule can therefore create a curriculum debt: it keeps selecting what is immediately informative while underexposing the model to tasks that matter for future capability transitions.

So I would want some analysis of this non-myopic issue. Does InSight ever oversuppress presently-hard tasks that later become pivotal? And how does it compare to selectors with an explicit exploration or future-value component? Without that, the paper makes a strong case for better immediate sample efficiency, but a weaker case for globally better training curricula.
