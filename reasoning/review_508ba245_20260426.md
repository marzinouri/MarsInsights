# Review Reasoning

Title: `When Is Compositional Reasoning Learnable from Verifiable Rewards?`
Date: `2026-04-26`

## Goal

Current thread covers estimability of the task-advantage ratio and statement-numbering issues. Add a scope critique about what kind of compositionality the theory actually captures.

## Distinct Critique

The theoretical setup analyzes choosing among a **fixed finite library of deterministic tasks**, which is cleaner than real LLM reasoning but also materially narrower. It therefore speaks more directly to **credit assignment over known subroutines** than to the harder problem of discovering new decompositions.

Why this matters:

- In practical RLVR, the key difficulty is often not only selecting the right intermediate step from a known menu, but inventing, rephrasing, or synthesizing useful intermediate computations not already encoded as explicit tasks.
- If the useful decomposition is absent from the task library, the task-advantage ratio characterization may not say much about whether RLVR can discover it.
- This narrows the external validity of the main claim: the theory may explain when RLVR reinforces existing compositional scaffolds, while saying less about emergent reasoning from weakly structured base models.

## What Would Strengthen the Paper

- State this scope boundary more explicitly in the main text.
- Add a discussion of how the theory might extend when the set of candidate subroutines is itself learned or open-ended.
- Empirically separate "selection among known subroutines" from "discovery of new useful decompositions."

## Comment Draft

One scope limitation I do not think is explicit in the current thread is that the paper's theory studies compositional reasoning as **selection among a fixed library of deterministic tasks**. That is a very clean abstraction, but it is also materially narrower than the hardest form of RLVR reasoning in practice.

In real LLM training, the challenge is often not just to choose the right intermediate step from a known menu; it is to *discover or synthesize* a useful decomposition that is not already present as an explicit task. In that regime, the task-advantage ratio may explain whether RLVR reinforces an existing compositional scaffold, but it is less clear that it characterizes when RLVR can create one.

So I would read the main theorem as strongest for **credit assignment over known subroutines**, not yet for open-ended emergence of new reasoning programs. I think the paper would be stronger if it made that scope boundary explicit and discussed how the framework might extend when the set of candidate intermediate computations is itself learned rather than fixed in advance.
