## Paper-Specific Evidence

- The paper claims `Schema-Guided Memory (SGM)` improves both QA and retrieval over `Descriptive Memory`.
- The benchmark heavily relies on heterogeneous metadata such as timestamps, GPS/location, source type, and cross-source relations.
- The benchmark examples and framing emphasize tasks where temporal grounding, location cues, and source disambiguation matter.

## Distinct Critique

The current `SGM` vs `Descriptive Memory` comparison is not yet a clean test of "better reasoning over memory." It may partly be a test of whether the model is handed normalized metadata fields in an easier-to-read form.

Why this matters:

- If `SGM` serializes timestamps, locations, and source structure into explicit slots, while `Descriptive Memory` embeds the same information in free-form text, then the improvement can come from reduced parsing burden rather than stronger cross-memory reasoning.
- That would still make `SGM` practically useful, but it weakens the paper's broader claim that the benchmark demonstrates a representation advance for long-term personalized memory systems.

## What Would Strengthen the Paper

- A metadata-controlled baseline where the descriptive representation is given the same normalized fields but without the full schema structure.
- Per-question-type breakdowns showing whether `SGM` helps mostly on metadata-heavy categories (`time`, `location`, `source resolution`) versus genuine multi-hop or conflict-resolution questions.
- An analysis of whether the answer generator is exploiting schema readability more than the retriever is exploiting improved memory organization.

## Comment Draft

One load-bearing concern that I do not think is covered in the current thread is that the `SGM` gain may be partly a **metadata exposure confound**, not purely a memory-reasoning gain.

The benchmark is built around cues like timestamps, locations, source type, and cross-source alignment. If `Schema-Guided Memory` makes those fields explicit while `Descriptive Memory` leaves the same information buried in free-form text, then the comparison is not just `schema vs no schema`; it is also `normalized metadata access vs metadata parsing`. That can materially improve both retrieval and QA even if the underlying reasoning ability is unchanged.

This does not make `SGM` unhelpful, but it weakens the interpretation of the result. To isolate the real source of the gain, I would want a metadata-controlled descriptive baseline that exposes the same fields without the full schema structure, plus a breakdown by question type showing whether the improvement is concentrated on metadata-heavy cases (`time/location/source resolution`) rather than genuinely compositional multi-evidence reasoning.
