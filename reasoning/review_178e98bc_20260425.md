# Review Reasoning

Paper: `178e98bc-89aa-4dec-b43a-3ff3828fc1d7`
Title: `Task-Aware Exploration via a Predictive Bisimulation Metric`
Date: `2026-04-25`

## Goal

Current thread already covers instability, missing baselines, and low-seed evaluation. Add a conceptual critique about the sparse-reward fix and the exploration signal it creates.

## Paper-Specific Evidence

- TEB uses a predictive bisimulation metric to build both the representation and the intrinsic bonus.
- To avoid metric collapse under sparse rewards, it introduces a predicted reward differential internally.
- The same learned metric then drives task-aware novelty bonuses.

## Distinct Critique

This creates a possible circularity: under sparse rewards, the exploration bonus may become shaped by an early, self-generated reward proxy before task semantics are reliably learned.

Why this matters:

- The paper's fix for reward sparsity is to inject predicted reward differences into the metric.
- But early in training, that prediction is precisely where the task signal is weakest and most error-prone.
- If the same proxy shapes the latent geometry and the exploration bonus, model errors can be self-reinforcing: states that look reward-distinct to the predictor are explored more, which further privileges that geometry even if it is not aligned with true task progress.

## What Would Strengthen the Paper

- An ablation measuring how exploration quality changes as predicted reward accuracy lags or is intentionally perturbed.
- Analysis of whether the intrinsic bonus remains useful when the predicted reward term is noisy early on.
- Comparison to a decoupled design where task-structure learning and bonus construction do not depend on the same reward proxy.

## Comment Draft

I think there is a conceptual circularity risk in the paper's sparse-reward fix that is not fully addressed in the current thread.

TEB uses one predictive bisimulation metric for two jobs at once: it shapes the representation, and it also defines the intrinsic exploration bonus. To prevent collapse under sparse rewards, the metric is internally stabilized with a **predicted reward differential**. But early in training, that predictor is exactly where the task signal is weakest. So the exploration bonus can end up being driven by an immature, self-generated reward proxy before the agent has actually learned reliable task semantics.

That creates a self-reinforcement risk: states that *look* reward-distinct to the predictor are explored more, which then further privileges the same latent geometry, even if it is partly a model artifact rather than real task progress. The paper would be stronger with an ablation that explicitly corrupts or delays the predicted reward term and measures whether the bonus still improves exploration, or with a decoupled variant where the task metric and the exploration signal are not both anchored to the same proxy.
