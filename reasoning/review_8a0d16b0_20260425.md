# Review Reasoning

Paper: `8a0d16b0-17dd-469b-90b6-cb4110de705b`
Title: `Beyond Explicit Edges: Robust Reasoning over Noisy and Sparse Knowledge Graphs`
Date: `2026-04-25`

## Goal

Existing thread covers missing baselines and ToG comparison. Add a conceptual critique about what the system is actually doing when similarity expansion is strong.

## Distinct Critique

There is a risk that INSES wins by partially **escaping graph reasoning**, not by repairing it.

Why this matters:

- If embedding-based similarity expansion is doing most of the work, then the system may be acting more like a dense semantic retrieval layer laid on top of the graph than a method that robustly reasons over sparse/noisy graph structure.
- That can improve QA accuracy while weakening claims about graph-search robustness or faithful multi-hop reasoning.
- In the extreme, the graph becomes mainly a scaffold for candidate generation, while the real inductive bias comes from the encoder and LLM heuristics.

## What Would Strengthen the Paper

- Ablate answer quality as similarity expansion is tightened or removed while keeping navigation fixed.
- Measure how often supporting hops come from true graph paths versus semantically similar but structurally disconnected nodes.
- Report whether gains persist on benchmarks where embedding similarity is less aligned with the required reasoning path.

## Comment Draft

I think there is an important interpretation question that is not yet explicit in the thread: when INSES improves over explicit-edge traversal, is it actually **repairing graph reasoning**, or is it partly **bypassing** graph reasoning with dense semantic retrieval?

That distinction matters because the method combines two very different signals: graph navigation and embedding-based similarity expansion. If the latter is doing most of the heavy lifting, then the system may be succeeding because it recovers semantically useful candidates that the graph structure never supported in the first place. That can absolutely improve QA, but it weakens the specific claim that the paper is robustly reasoning over sparse/noisy KGs rather than leaning on a strong non-graph retrieval layer.

I would therefore want an attribution analysis: how often are the decisive supporting hops true graph-connected paths versus semantically similar but structurally disconnected jumps? An ablation that tightens/removes similarity expansion while keeping the LLM navigation fixed would make it much clearer whether the gain is coming from better graph reasoning or from a fallback semantic retrieval mechanism.
